# subject-weave

학생 글에서 뽑은 문장, 자기평가서, 교사 관찰 — **세 갈래를 엮어** 학생별 교과 세특
프롬프트를 만들고, AI가 쓴 초안의 바이트를 재는 도구.

> 앞 단계는 [`pdf-splitter-plus`](../../../pdf-splitter-plus) — 스캔 PDF를 학생별로 나누고
> 판독 지시문을 준다.

## 이 앱이 하지 않는 것

**세특을 쓰지 않는다.** 판독도, 평가도, 문장 다듬기도, 분량 줄이기도 하지 않는다.
그건 전부 교사와 교사의 AI 몫이다.

이 앱이 없애는 것은 **지능이 필요 없는 반복 노동**뿐이다 — 자기평가서를 25번 복사해
붙이기, 같은 지시문을 25번 타이핑하기, 나이스 카운터에 25번 붙여넣기.

## 파이프라인

```
① pdf-splitter-plus  → 학생별 PDF + 판독 지시문
② 각자의 AI (Gemini) → 주제 + 핵심 문장 추출 (### 이름 형식)
③ subject-weave      → ②결과 + 자기평가 엑셀 + 교사 관찰 → 학생별 프롬프트
④ 각자의 AI          → 세특 초안
⑤ subject-weave      → 바이트 검증 + 엑셀 출력
```

③⑤가 이 앱이다. 조립은 텍스트 작업이라 **어떤 AI든 된다**(판독과 달리).

## 화면

1. **자기평가 엑셀** — 리로스쿨 원본 그대로. 꺾쇠 제거 불필요
2. **추출 결과 붙여넣기** — `### 이름` 단위로 갈라 학생과 맞춤
3. **교사 관찰과 문체** — 학생별 키워드 + 문체 3택 → 프롬프트 복사
4. **바이트 검증** — 초안을 붙여넣으면 재고 표시. 엑셀 출력

## 알아둘 것

**꺾쇠 제거가 필요 없다.** 리로스쿨 엑셀은 가이드 항목마다 열을 따로 뽑아준다
(`[선택] …`, `[필수] …` 헤더). 그 열을 그대로 읽으면 `글내용(통합)`에서 꺾쇠를
지운 것과 **정확히 같은 결과**가 나온다(검증 완료). 항목 열이 없는 옛 엑셀이면
자동으로 꺾쇠 제거 방식으로 넘어간다.

**항목 열이 통합 열보다 낫다.** 어느 답이 어느 항목인지 살아 있어서, 프롬프트에
`[필수]`/`[선택]` 표식을 붙여 넘길 수 있다.

**항목 칸을 무시하고 한 칸에 몰아 쓰는 학생이 있다.** 감지해서 카드에 표시하고,
프롬프트에 "항목 구분 없이 전체를 읽어라"를 넣는다.

**교사 관찰은 어디에도 저장되지 않는다.** 리로스쿨에 쓰면 학생이 보고, 파일로
남기면 유출된다. 브라우저 메모리에만 있다가 프롬프트에 실려 나가고 새로고침하면
사라진다. 이 앱이 서버 없이 만들어진 이유다.

**AI 결과를 긁어서 복사하지 말 것.** 답변 아래 복사 버튼(⧉)을 쓴다. 긁으면 `###`과
줄바꿈이 사라져 `김서윤주제:`처럼 뭉친다. 그래도 파싱하도록 만들었지만, 복사 버튼이
정확하다.

**이름은 자주 틀린다.** AI가 손글씨에서 이름을 잘못 읽는다(김지후→김기후,
최민송→최민용). 한 글자 차이면 후보를 제안하고, 드롭다운으로 직접 지정할 수 있다.

## 프롬프트에 박은 규칙

- 교사 관찰 문체, 종결 `~함`/`~음`, 1인칭 제거
- **평가 대상 어법은 절대 바꾸지 않는다** — 수동태를 능동태로, 가정법을 직설법으로
  바꾸면 평가 대상이 사라진다
- 뜻을 살리는 최소한의 교정만. 더 멋진 문장으로 새로 쓰지 않는다
- 사실(제안 내용, 주제, 활동명, 진로 분야)은 불변
- 인용은 작은따옴표, 원문에 없던 따옴표를 새로 씌우지 않음
- `[판독불가]`가 있는 문장은 인용 금지
- **1450은 상한이지 채워야 할 분량이 아니다.** 자료가 적은 학생을 억지로 늘이지 않는다
- 부족한 점은 완곡하게, **단 사실을 뒤집지 않고**, 그 학생의 활동에 맞춰 **새로** 쓴다
  (예시를 주면 AI가 그대로 베껴 20명이 같은 문장을 받는다 — 실제로 겪었다)

## 바이트 계산

나이스 기준(`hjh010501.github.io/neis-counter/`): **한글 3바이트, 줄바꿈 2바이트,
나머지 전부 1바이트.** 작은따옴표 `'` `'` 같은 문자는 비ASCII지만 1바이트다 —
"비ASCII = 3바이트"로 뭉뚱그리면 인용문 하나당 4바이트씩 밀린다(실제로 최대 20바이트
어긋났다). 최가경 세특으로 대조해 1453 일치 확인.

## 아직 확인 안 된 것

- **묶음 복사의 안전선.** 한 번에 몇 명까지 AI가 안 섞는지 안 재봤다. 5명 안팎 권장은
  추측이다.
